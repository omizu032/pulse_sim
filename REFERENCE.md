# 技術仕様書 (Technical Reference)

## 1. パルス出力仕様
* **出力ピン**: D10 (ATmega328P: PB2 / OC1B)
* **極性**: Active-HIGH (普段は0V、トリガー時に5Vを出力)
* **パルス幅**: 40μs 
  * ※定数 `PULSE_WIDTH_US` にて4単位（16, 32, 100μsなど）で設定すること。
* **対応回転数**: プログラム上の安全下限値は 100 RPM（※プリスケーラ64設定時、タイマーのオーバーフロー限界は実質 230 RPM付近）。上限はマイコンの処理能力に依存（実用上 10000 RPM 以上も安定出力可能）。

## 2. Timer1 (ハードウェアタイマー) レジスタ設定
パルスの生成には16bitタイマーである `Timer1` の **Fast PWM モード (Mode 15)** を使用しています。

* **TCCR1A / TCCR1B 設定**:
  * `WGM13:10 = 1111` (Fast PWM, TOP = OCR1A)
  * `CS11:10 = 11` (プリスケーラ 64)
  * `COM1B1:0 = 10` (BOTTOMでHIGHセット、OCR1BマッチでLOWクリア)
* **時間分解能**:
  * システムクロック 16MHz / 64 = 250kHz
  * 1カウント = 4マイクロ秒 (4μs)
* **パルス幅制御**:
  * `OCR1B = 40μs / 4μs = 10` (カウント値)
* **周波数制御 (RPM変換)**:
  * 周期T(秒) = 60 / RPM
  * TOP値 `OCR1A` = `(15,000,000 / RPM) - 1`

## 3. ソフトウェアアルゴリズム

### 3.1 ダブルバッファリングによるジッター防止
`OCR1A` レジスタへの書き込みはメインループ内で非同期に行われますが、ATmega328PのFast PWMモードの仕様により、レジスタ値の更新は「現在のPWM周期が完了した瞬間（BOTTOMに達した時）」にハードウェアレベルで同期的に実行されます。これにより、計算タイミングのズレによる波形のグリッチ（異常な短縮や延長）が発生しないようにしています。

### 3.2 マニュアル入力のノイズ除去 (ヒステリシス)
ADC入力（A0）は±3のヒステリシスを持たせて評価され、さらに `map()` 関数によって 0〜1023 の値を 10〜100 に丸めた後、100倍することで「100 RPM単位」の離散値に量子化しています。これにより、VRから手を離した際の環境ノイズによる計算上のRPMの微細な揺らぎを少なくしています。

### 3.3 オートスイープの線形補間
スイープモードにおける RPM の算出は、`millis()` による経過時間を用いた一次方程式（線形補間）によって行われます。
* `RPM = startRPM + ((endRPM - startRPM) * elapsed) / sweepDuration`
計算オーバーフローを防ぐため、内部計算は `long` 型にキャストして実行されます。

## 4. 拡張パラメータ
定数を変更することで、柔軟な設定に変更可能です。

* `sweepDuration`: 片道のスイープにかける時間（ミリ秒）
* `sweepMinRPM` / `sweepMaxRPM`: スイープの下限・上限回転数
* `startSweepFromMin`: `true`でMinから開始、`false`でMaxから開始
* `sweepRepeatCount`: 往復スイープを実行する回数
